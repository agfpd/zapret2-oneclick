# zapret2-oneclick

Готовая сборка zapret2 для Windows 10/11 x86_64. Пользователь уже знает, для
чего она ему нужна; ниже только установка, проверка и удаление.

## Установка

1. Полностью распакуйте ZIP в обычную папку. Не запускайте файлы прямо из
   окна архиватора.
2. Временно закройте VPN и другие программы, которые меняют сетевой трафик.
3. Нажмите правой кнопкой мыши на `setup.cmd`, выберите **«Запуск от имени
   администратора»** и подтвердите запрос Windows.
4. Не закрывайте окно. Первый автоматический подбор может занять несколько
   минут; каждые 15 секунд программа сообщает, что работа продолжается.
5. Дождитесь строки **`Installed successfully`**. После неё окно можно закрыть,
   перезагрузка не нужна.

## Проверка и обслуживание

Откройте нужные сайты и приложения обычным способом. Если результат не
изменился, запустите повторный подбор из распакованной папки:

```bat
setup.cmd -Rescan
```

Полезные команды:

```bat
setup.cmd -Rescan
setup.cmd -Rollback
uninstall.cmd
uninstall.cmd -KeepLogs
```

- `-Rescan` — заново подобрать настройки;
- `-Rollback` — вернуть предыдущие рабочие настройки;
- `uninstall.cmd` — удалить службу и файлы программы;
- `-KeepLogs` — при удалении сохранить журналы диагностики.

Журналы находятся в
`%ProgramData%\zapret2-oneclick.logs\runtime\logs`.

## SmartScreen и антивирус

Для установки службы и сетевого драйвера нужны права администратора. Windows
SmartScreen может показать предупреждение для скачанного `.cmd`-файла:
нажмите **«Подробнее» → «Выполнить в любом случае»**, только если архив получен
из доверенного источника и его SHA-256 совпадает с контрольной суммой релиза.

WinDivert работает с сетевым трафиком на уровне системы, поэтому антивирус
может посчитать его подозрительным. Не отключайте защиту вслепую и не заменяйте
файлы версиями из сторонних зеркал. Если антивирус удалил файл, сохраните
название сработавшего правила и сообщите сопровождающему сборки.

## Для разработчиков

В сборке используются официальный zapret2 v1.0.2, WinDivert, Lua-стратегии и
переносимая среда Cygwin 3.6.9. Вложенные бинарники проверяются по SHA-256.

Перед упаковкой выполняются:

```powershell
.\build\check-release-readiness.ps1
.\build\Build-SourceArtifact.ps1
```

Рядом с бинарным ZIP поставляется архив соответствующих исходников. Приёмка
всегда проводится из готового ZIP: его распаковывают в новую папку на чистой
Windows-машине и запускают `setup.cmd` точно так же, как конечный пользователь.
